- 添加一个模式选择，
  - 一种模式是日常聊天
  - 另一种是工作模式:
    - 支持将任务拆分为多轮循环
    - 每一轮包含:
      - `分析`
      - `生成或调用工具执行任务`
      - `检验结果`
      - `总结`: 如果结果不满意，则调整提示词、工具调用参数等然后重新尝试本轮
        - （本轮不满意或失败时，应该总结原因将这一轮的message压缩成条，但这一轮的message不丢弃，而是可以被用户查看，只是不发送到模型推理）
        - （每一轮开始时，应当总结出一条包含任务内容提示、前面已完成的任务、待完成务、注意事项，然后带着这一条message开始本轮任务执行，而不是把其他轮次的message也发送给ai推理）
    - 对于工作模式，AI收到用户任务后，应当：
      - 先自动分析、尝试拆分、决定结果的检验方法和标准
      - 然后把整体方案再展示给用户，用户可提出修改意见
      - 最终确认方案后AI开始执行
    - 工作模式实现应该结合langgraph，以可能需要的提示词、skill 去实现，确保稳定性
